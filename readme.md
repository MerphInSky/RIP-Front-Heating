Цель работы: Подключение бэкенда и добавление модели CLIP
Порядок показа: Показать сначала три страницы фронтенда с mock без запущенного сервиса, а затем показать страницы фронтенда с бэкендом, показать в network адрес fetch через proxy, названия и адреса из minio для изображений. Внести изменения в БД, показать их во фронтенде. Объяснить вызовы fetch.
Контрольные вопросы: CORS и обратный прокси, BFF и GraphQL, Next.js и SSG, FSD
Задание: необходимо для 3 запросов GET для двух страниц приложения и реализовать мультимодальный поиск через CLIP
Содержимое карточек получать из вашего веб-сервиса. Ajax-запросы написать самостоятельно через fetch. Ограничение с CORS решить через проксирование React. В методах fetch предусмотреть получение данных из коллекции с mock-объектами при отсутствии доступа к вашему бэкенду. Фильтры (по дате, названию или цене в зависимости от вашей темы как в 1 и 3 лабораторных) также добавлять в запрос к бэкенду. Должны быть использованы 3 запроса GET: список услуг с фильтрацией, одна услуга и иконка корзины (без авторизации, но ответ 200)

На странице Список услуг добавить возможность загрузки файла изображения для поиска и использовать его для поиска похожих карточек на основе близости эмбеддингов описаний на английском (краткие описания в 50-100 символов строго на английском), для вычисления эмбеддингов использовать transformer.js.
